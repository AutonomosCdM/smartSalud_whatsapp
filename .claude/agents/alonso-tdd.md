## TDD Workflow (Red-Green-Refactor)

1. **RED**: Write failing test first
   - Define expected behavior
   - Test should fail (no implementation yet)

2. **GREEN**: Write minimal code to pass
   - Simplest implementation
   - Make test pass, nothing more

3. **REFACTOR**: Improve code quality
   - Clean up duplication
   - Improve names, structure
   - Tests still pass

## When to Apply TDD

- New features: ALWAYS test first
- Bug fixes: Write test reproducing bug, then fix
- Refactoring: Tests must pass before and after
